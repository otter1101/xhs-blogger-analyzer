## 🎯 它解决了什么问题？

**小红书创作者的核心痛点：**

你想对标一个做得好的博主，学习 TA 的内容策略。但——

- 🥱 **手动翻笔记太慢** — 50 条笔记逐条看完要一整个下午，看完也记不住规律
- 🤷 **看了也分析不出来** — 知道 TA 赞多，但说不清为什么多、你该怎么学
- 📊 **缺数据支撑** — "感觉 TA 最近在转型"，但到底是从什么转到什么？藏赞比说明什么？
- 🔁 **每次都从零开始** — 拆解完一个博主，换一个又要重复所有步骤

**xhs-blogger-analyzer 把这件事从「手工活」变成了「一句话的事」。**

```
拆解博主 （对标博主账号）
```

等20分钟，拿到 4 份 Word 文档，包含完整的内容策略拆解。

## ✨ 功能特性

### 核心能力

- 🤖 **全自动流水线** — 环境准备 → 扫码登录 → 数据爬取 → 统计分析 → AI 深度解读 → Word 文档输出，一条龙
- 🧠 **脚本 + AI 双引擎** — 脚本做脏活（爬数据/算统计），AI 做洞察（拆人设/提公式/给建议），各司其职
- 📊 **数据驱动分析** — 11 种标题模式识别、6 类 CTA 提取、藏赞比三档分类、发展趋势追踪
- 📝 **4 份专业文档** — 博主拆解 / 内容公式 / 选题素材库 / 结构化分析，直接可用
- 🔄 **对比分析** — 支持"拆解 TA + 对比我"模式，输出针对性改进建议
- 🌍 **全领域通用** — 不预设领域标签，基于笔记标签动态聚类自动适配

### 技术亮点

- 🔧 **零配置启动** — 自动下载依赖、自动启动服务，用户只需扫码登录
- 🛡️ **断点恢复** — 每 10 条笔记自动存档，网络抖动不丢数据
- 📱 **扫码降级策略** — 渲染图片 → 保存 PNG 到桌面 → 输出 URL，三级降级保证可用
- 🔌 **纯标准库** — MCP 客户端只用 `urllib`，不依赖 `requests`，减少安装摩擦
- 📄 **中文排版优化** — Word 文档使用微软雅黑、1.35 行距、交替行表格着色

## 🚀 快速开始

### 安装

```bash
# 克隆项目
git clone https://github.com/your-username/xhs-blogger-analyzer.git
cd xhs-blogger-analyzer

# 自动安装到 AI 编程助手（自动检测 WorkBuddy / Claude Code）
python install.py
```

### 两种使用模式

#### 模式一：自然语言触发（推荐）

安装完成后，在你的 AI 编程助手中直接说：

```
拆解博主 （对标博主账号）
```

AI 自动执行完整流程。你只需要在第一次使用时扫码登录小红书。

**更多玩法：**

```
拆解博主 XX，对比我的账号 （我的账号）
拆解美食博主 XX，关键词：烘焙、食谱、探店
依次拆解以下博主：XX、YY、ZZ
```

#### 模式二：命令行一键运行

```bash
# 基本用法
python run.py "（对标博主账号）"

# 带对比分析（和自己的账号对比）
python run.py "（对标博主账号）" --self "（我的账号）"

# 指定领域关键词（优化搜索覆盖率）
python run.py "（对标博主账号）" --keywords "烘焙,食谱,探店"
```

首次运行时，Skill 会自动：
1. 安装 `python-docx` 依赖
2. 从 GitHub 下载 `xiaohongshu-mcp` 二进制
3. 启动 MCP 服务
4. 弹出二维码 → 手机扫码登录

**之后再运行就是一句话直出结果。**

## 📦 产出物展示

每次分析产出 **4 份 Word 文档**，以下是真实产出物截取：

### 📄 文档一：博主深度拆解

> 深入理解对标博主的全貌——TA 是谁、做了什么、为什么好、你能学什么。

**包含内容：**
- 账号画像（粉丝量/获赞/内容定位/人设标签）
- 人设三板斧拆解（粉丝跟 TA 的本质原因）
- TOP10 高赞笔记逐条拆解（标题策略 + 爆款原因 + 评论洞察）
- 内容模式分析（垂直型 vs 多元交叉型）
- 评论区"下一步内容机会"提取

### 📄 文档二：内容公式总结

> 把对标博主的套路提炼成你可以直接套用的模板。

**包含内容：**
- 11 种标题公式（数字型/疑问型/感叹型/对比型/降门槛型/亲测型……）
- 开头公式 / 结构模板 / CTA 策略 / 视觉排版公式
- 每种公式附改编示例（针对你的账号）

### 📄 文档三：选题素材库

> 直接告诉你接下来可以发什么。

**包含内容：**
- 可借鉴选题清单（按优先级 × 创作难度排序）
- 差异化赛道机会
- 系列 IP 建议
- 已做选题对照表（有对比数据时）

### 📄 文档四：全量笔记结构化分析

> 数据驱动的底层规律挖掘。

**包含内容：**
- 领域分布 & 藏赞比三档分析（实用型/均衡型/情绪型）
- 爆款公式总结（阈值、集中领域、形式分布）
- 发展趋势 & 条件性转型路径分析
- 竞争格局分析

> 已在 AI、职场成长、女性成长、效率工具等多个赛道实测验证，笔记数从 50 到 70+ 条均可稳定产出。

## 🏗️ 架构设计

### 流水线架构

```
Phase 0          Phase 1          Phase 2          Phase 3        Phase 3.5
环境自动准备 ──→ 数据采集 ──────→ 数据分析 ──────→ 骨架文档 ──→ AI 深度分析
                                                                     │
┌─────────┐   ┌───────────┐   ┌───────────┐   ┌──────────┐   ┌──────┴──────┐
│ check_  │   │ crawl_    │   │ analyze   │   │ generate │   │ deep_       │
│ env.py  │   │ blogger.py│   │ .py       │   │ _docs.py │   │ analyze.py  │
│         │   │           │   │           │   │          │   │      +      │
│ • python│   │ • 搜索博主│   │ • 动态聚类│   │ • MD骨架 │   │  AI Engine  │
│ • docx  │   │ • 主页信息│   │ • 标签统计│   │ • Word   │   │             │
│ • MCP   │   │ • 多关键词│   │ • TOP10   │   │   转换   │   │ • 人设拆解  │
│ • 登录  │   │ • 笔记详情│   │ • 对比    │   │          │   │ • 因果分析  │
│         │   │ • 断点恢复│   │           │   │          │   │ • 公式提炼  │
└─────────┘   └───────────┘   └───────────┘   └──────────┘   │ • 个性建议  │
                                                               └─────────────┘
                                                              脚本30% + AI 70%
```

### 核心设计理念

**"脚本保下限，AI 冲上限"**

| 角色 | 占比 | 负责内容 |
|------|------|---------|
| **脚本** | 30% | 数据采集、统计计算、模式识别（11种标题/6类CTA/Emoji/发布频率） |
| **AI** | 70% | 人设拆解、因果分析、公式提炼、个性化建议、评论洞察、金句总结 |

为什么这样分？因为"统计一个博主有多少篇笔记用了数字标题"是确定性工作，脚本可以 100% 准确；但"这个博主为什么能涨粉"是需要推理和洞察的，AI 做得更好。

### MCP 通信

```
AI 编程助手 ←→ Skill (SKILL.md + Python 脚本) ←→ MCP Client (urllib)
                                                         ↕
                                                   xiaohongshu-mcp
                                                    (localhost HTTP)
                                                         ↕
                                                      小红书 API
```

使用 [xiaohongshu-mcp](https://github.com/anthropics/xiaohongshu-mcp) 作为数据桥梁，通过 HTTP JSON-RPC 协议通信，纯标准库实现（`urllib`），零外部依赖。

## 📋 AI 分析质量准则

为保证 AI 产出的分析有洞察、有深度、不敷衍，我们定义了 **8 条写作准则**：

| # | 准则 | 一句话说明 |
|---|------|---------|
| 1 | **有观点不骑墙** | 每个章节必须有明确判断，用加粗标出 |
| 2 | **有对比有洞察** | 产量 vs 效果、早期 vs 近期、爆款 vs 低赞 |
| 3 | **有针对性建议** | 如果提供了用户账号，给出可执行的改进建议 |
| 4 | **有因果解释** | 不只说"赞多"，要说清楚为什么赞多 |
| 5 | **有层次感** | TOP5 深度拆解，TOP6-10 快速拆解 |
| 6 | **有金句记忆点** | 提炼"博主底层公式"，章节结论加粗 |
| 7 | **有数据锚点** | 禁止"表现较好"，必须跟具体数字 |
| 8 | **格式有节奏** | 数据=表格、分析=段落、结论=引用块 |
