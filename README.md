## 博主蒸馏器做了什么

把「手工翻笔记 → 凭感觉总结」变成「一句话 → 自动蒸馏 → 拿结果」。

```
拆解/蒸馏小红书博主 XX
```

等 30–60 分钟。AI 自动完成：

```
扫码登录 → 爬取 50 条笔记 → 统计分析 → 认知层提取 → 蒸馏 → 出报告
```

你拿到的不是一堆原始数据，而是：

| 产出物 | 给谁看 | 干什么用 |
|--------|--------|----------|
| **HTML 蒸馏报告** | 给你 | 浏览器打开，快速理解这个博主的人设、策略和爆款规律 |
| **创作指南 Skill 文件夹** | 给你的 AI | 安装后说"用 XX 风格写一篇笔记"，AI 立刻知道怎么写 |

> **和竞品的区别**：市面上的蒸馏类 Skill（同事.skill / 前任.skill / 女娲.skill）都需要你手动提供数据——聊天记录、文档、截图。博主蒸馏器是目前唯一**输入端全自动**的蒸馏类skill：你只需要说一个博主的帐号。

## 三层蒸馏：不只是统计，是提炼思维方式

大多数分析工具止步于"这条笔记赞多"。博主蒸馏器往下挖三层：

| 层级 | 回答什么问题 | 举例 |
|------|-------------|------|
| **认知层** | TA 怎么想？核心信念是什么？ | "TA 相信'反共识才有传播力'，80% 爆款都用了反常识观点开头" |
| **策略层** | TA 怎么运营？什么时候发？蹭什么热点？ | "每周三晚 8 点发，藏赞比 0.85 说明粉丝当教程收藏" |
| **内容层** | TA 怎么写？标题公式？开头模板？CTA？ | "35% 数字标题、25% 反问标题，开头永远是个人故事" |

脚本负责保下限（爬数据、算统计），AI 负责冲上限（提炼信念、发现规律、给你建议）。

## 产出物

### HTML 蒸馏报告（给你看）

一份单文件 HTML，浏览器打开即可阅读，不需要安装任何东西。

包含 10 个模块：

| # | 模块 | 一句话说明 |
|---|------|-----------|
| 1 | 一眼看清 | 粉丝/获赞/爆款率/底层公式，一张卡片看完 |
| 2 | 人设拆解 | TA 是谁、代表什么、粉丝为什么追 TA |
| 3 | 认知层 | 核心信念 × 观点张力 × 价值立场 |
| 4 | 策略层 | 系列规划 × 蹭热点方式 × 运营节奏 |
| 5 | TOP10 爆款 | 逐条拆解：为什么爆？能学什么？ |
| 6 | 内容公式 | 11 种标题公式 × 开头模板 × CTA 策略 |
| 7 | 选题灵感 | TOP15 选题方向，按难度 × 潜力排序 |
| 8 | 数据面板 | 均赞/均藏/藏赞比/视频 vs 图文/发布频率 |
| 9 | 发展趋势 | 早期 vs 近期变化，转型路径（附置信度标注） |
| 10 | 核心结论 | 三句话总结 + 行动建议 |

### 创作指南 Skill（给 AI 用）

一个可安装的 Skill 文件夹。生成后，把整个文件夹放到以下任一目录：

- 项目级：`.claude/skills/`
- 全局：`~/.claude/skills/`

```
.claude/skills/XX博主_创作指南.skill/
└── SKILL.md
```

三层蒸馏结构：
- **认知层**：像 TA 一样思考（信念 + 张力 + 思维模式）
- **策略层**：像 TA 一样决策（系列规划 + 运营规则 + 标签策略）
- **内容层**：像 TA 一样写（标题公式 + 开头模板 + CTA + 视觉风格）

安装后，在新对话里直接说：

```
用XX博主的风格，写一篇关于 AI 工具推荐的小红书笔记
```

AI 会按照蒸馏出的公式、语气和结构来写——不是泛泛的"活泼风格"，是从 50 条真实笔记里提炼的具体规则。

## 30 秒上手

### 安装

```bash
git clone https://github.com/otter1101/xhs-blogger-analyzer.git
cd xhs-blogger-analyzer
python install.py
```

### 开始蒸馏

在你的 AI 编程助手里说：

```
拆解博主 XX
```

系统会问你两件事：

1. **选模式**：A（蒸别人的账号）还是 B（蒸自己的账号）
2. **选数量**：30 条（快速）/ 50 条（推荐）/ 80 条（深度）

然后就是全自动了。第一次需要扫码登录小红书，之后不用。

## 架构设计

### 「脚本保下限，AI 冲上限」

```
Phase 0          Phase 0.5       Phase 1        Phase 2              Phase 3
环境自动准备 ──→ 前置交互 ──→ 数据采集 ──→ 分析+认知层提取 ──→ 蒸馏+产出物生成
                 (模式/数量)                                         │
                                                              ┌─────┴──────┐
                                                              │  Step A    │
                                                              │ 脚本：数据 │
                                                              │ 底稿+蒸馏  │
                                                              │ 任务       │
                                                              ├────────────┤
                                                              │  Step B    │
                                                              │ AI：HTML   │  → Phase 4
                                                              │ 报告+Skill │  → 质量校验
                                                              │ 文件夹     │
                                                              └────────────┘
```

| 角色 | 占比 | 负责什么 |
|------|------|---------|
| **脚本** | 30% | 爬数据、算统计、识别 11 种标题模式、提取 CTA、计算藏赞比 |
| **AI** | 70% | 提炼信念、发现认知张力、写人设拆解、做因果分析、给个性化建议 |

> 为什么这样分？"统计多少篇笔记用了数字标题"是确定性工作，脚本 100% 准确。但"这个博主为什么能涨粉"需要推理和洞察，AI 做得更好。

### 核心技术特性

- **零配置** — 自动下载依赖、启动服务，你只管扫码
- **断点恢复** — 每 10 条自动存盘，断网不丢数据
- **扫码三级降级** — 渲染图片 → 保存 PNG 到桌面 → 输出 URL
- **纯标准库** — MCP 客户端只用 `urllib`，不依赖 `requests`
- **全领域通用** — 不预设标签，基于笔记内容动态聚类

## 项目结构

```
xhs-blogger-analyzer/
├── SKILL.md                  # Skill 定义（AI 编程助手读这个）
├── run.py                    # 一键运行入口
├── install.py                # 自动安装脚本
├── scripts/
│   ├── check_env.py          # Phase 0: 环境准备（下载/安装/启动/登录）
│   ├── crawl_blogger.py      # Phase 1: 数据采集（搜索+主页+详情+断点）
│   ├── analyze.py            # Phase 2: 数据分析（聚类+标签+TOP10+认知层）
│   ├── deep_analyze.py       # Phase 3: 数据底稿 + AI 蒸馏任务生成
│   ├── verify.py             # Phase 4: 数据校验
│   └── utils/
│       ├── mcp_client.py     # MCP HTTP 客户端（纯 urllib）
│       ├── md_to_docx.py     # 历史兼容工具
│       └── common.py         # 共用工具函数
└── references/
    └── 产出物质量标杆.md      # 产出结构与质量参考
```
